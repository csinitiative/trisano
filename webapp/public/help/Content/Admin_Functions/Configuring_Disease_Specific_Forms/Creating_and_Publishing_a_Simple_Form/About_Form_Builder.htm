About Form Builder

CSI TriSano™ includes a Form Builder tool that allows you to extend its capabilities by capturing additional information for existing or new public health situations. Form Builder can be extremely useful during a disease outbreak, such as the recent H1N1 influenza virus. It allows you to respond quickly to emerging or changing health events by allowing you to create customized forms.

Form Builder lets you capture additional information for each of the different event types including:

  • Morbidity events (CMRs)
  • Contact events
  • Encounter events
  • Place Exposures

Adding Questions

Form Builder allows you to add questions to the existing tabs of a CMR. Unless otherwise specified, questions added using Form Builder appear on the Investigation tab for the morbidity or contact event. You can also add questions:

  • To existing tabs of an event, such as the Demographic, Clinical, or Laboratory tabs
  • Before or after existing questions on a tab
  • As follow up questions that appear based on the answer to a question you have defined or that already exists on one of the tabs

Considering Different Types of Forms

Forms can be simple or they can be very complex. In fact, you can simplify existing paper disease investigation forms when you automate them with Form Builder since repetitive data only needs to be entered once and optional information is requested only when required. In addition, you can add help text to any question created using Form Builder to guide the investigator. In addition, you can:

  • Define forms for diseases for one or all jurisdictions
  • Copy form elements to the Form Library for use when creating other forms
  • Activate or deactivate forms
  • Copy forms
  • Add forms to events that already exist in the system
  • Remove forms from events
  • Export forms to another CSI TriSano™ system
  • Import forms from another CSI TriSano™ system

Creating a Form 

To create and publish a form you must:

  1. Identify form elements to be defined – The application already contains a large amount of demographic, clinical, epidemiological, and other data. Consider only additional data that you will need to define in Form Builder.
  2. Create the form using Form Builder – First, you must define basic information about the form such as form name, diseases it applies to, and jurisdictions where the form can be used. Next, you define the questions and possible answers for additional information that is not already part of the application.  Then, you specify the location of each question or group of questions within the tabbed interface.
  3. Publish the form in Form Builder – This makes the form available to the application and allows you to use it with new events. If necessary, you can add the new form to events already in the system. Once you publish the form, it is automatically assigned to any new event that is created in the system for any disease that the form belongs with.
