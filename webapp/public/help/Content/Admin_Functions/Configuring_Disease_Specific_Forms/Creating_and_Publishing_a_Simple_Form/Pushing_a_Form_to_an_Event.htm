Pushing a Form to an Event

The Push feature assigns a form to existing events. It is an efficient and time-saving way to assign a form to many events all at the same time. For example, the database already has 100 events for patients with measles. The CDC sends you a new form containing questions that must be included in measles events. You must create the form and then add it to the events of all 100 patients. You can do that one event at a time (see Assigning a Form to an Event) or you could add it to all 100 events at one time using Push.

The Push link appears on the main Forms page and on the Builder and Edit pages. Use Push when a form, typically a new or revised form, has been created and must be assigned to a large group of events. Publishing a form does not assign it to existing events automatically. 

Consider the following example:

  • There is one existing measles form.
  • There are 40 morbidity events for measles in the system; the Monkeypox form has been assigned to all 40 events and all the questions have been answered.
  • You receive new measles form from the CDC, so you Create the new form using Form Builder and Publish it.
  • This newly published form will be assigned to new measles morbidity events, but it will not be automatically assigned to the 40 morbidity events that already exist for the measles.
  • If you want this new form to be attached to the already existing events, use Push.
  • Pushing does not assign the form to any events that already have a version of this particular form assigned.
  • If you need to access all the measles events that have the new form assigned, you can use Search and create a list of measles events to be edited and updated.

To assign the a form to all events for a selected disease that do not already have this form assigned:

  1. Click Admin in the upper right corner of the page.
  2. Click Manage Forms under Disease Specific Forms. The Forms list opens displaying the forms in alphabetical order.
  3. To push this form to the events for a selected disease, located the form in the list and click Push in the same row as the disease. A message at the top of the screen tells you that the Form was successfully pushed to events. This means that the all events that have been assigned the selected disease for which the form was created, have now been assigned this form.

To verify that the form has been assigned to one of the events:

  1. Click Events in the upper right corner of the page. The Events List opens.
  2. Locate an event that has the selected disease.
  3. Click Edit.
  4. Select the Investigation tab. Verify that the form has been assigned to the event.
