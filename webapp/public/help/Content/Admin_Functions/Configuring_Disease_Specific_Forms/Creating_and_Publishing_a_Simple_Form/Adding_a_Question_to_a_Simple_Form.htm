Adding a Question to a Simple Form

Questions are a very useful tool in collecting information. You can customize the placement and wording of questions to help get to the data you need.

Using the Builder

After you have created a simple form and entered basic information, you can use the Builder to add any number of questions to the form. 

In addition, you can add a question and based on a specified answer to that question, add follow up questions.

Gathering information using such custom questions helps you record data that is required by the form, but that is not a built-in part  (core field) of CSI TriSano™.

Understanding Core Fields

We often refer to pieces of data called core fields. Core fields are the data fields that are built-in to CSI TriSano™. They are what already exists in CSI TriSano™ before you starting adding questions and selections for answers.

Launching the Builder

Click Builder to begin adding questions and sets of answers to the form. The Builder page opens.

The Builder page lets you create forms. Buttons along the top of the page allow you to add questions to different parts of events. The following table describes the buttons.

 

Button Function
Add Tab Allows you to add a tab under the Investigation tab of an event. You can then add questions under that tab and add any number of additional tabs.
Add Core Tab Config Allows you to add questions to an event's existing tabs: Demographic, Clinical, and other tabs. The questions are placed at the end of the core fields on that tab.
Add Core Field Config Allows you to add questions before or after an event's core fields. You can add questions that are based on the answer to that core field or any other core field.
Open Library Allows you to view questions and answers (or value sets) in the library. You can add questions and answer to the library and determine how to use them to standardize and streamline development of forms.

 

A gray bar called Default View appears in the middle of the page. The Default View displays questions that you create if you have not specified an alternate location, such as before or after a core field. Questions listed under Default View appear on an event's Investigation tab. The following table explains the choices that may be available to you as you create a form.

 

Button Function
Add Section to Tab Allows you to add a section to a tab. The section can have a heading, instructions, help text and questions.
Add Question to Tab Allows you to add questions to the tab. In this case, the questions will be added to the Default View, the Investigation tab.
Trash Can Icon Allows you to delete that part of the form.
Publish Publishes the form. It is then available for use.

Adding a Question

To add a question to the Default View (Investigation tab):

  1. Click Add question to tab. The Default View section of the page expands and allows you to enter and update questions. The following table describes the functions available on this page.

 

Option Description
Question text Allows you to enter the text of the question.
Short name Allows you to enter a short name for the question. Short names must be unique within the form you are defining.
Is active? Determines if the question appears on the form. The default is Yes.
Style

Determines the location of the question:

  • Horizontal means that the next question will be placed on the same line as this question.
  • Vertical means that the next question will be place under this question.
Help text Allows you to add help text for a question. When you use this form, a question mark appears next to the entry field for the question. When you hover the mouse pointer over the question mark, a pop-up window with help text appears.
CDC Export column Allows you to link this question with a previously defined CDC Export column. If you select a CDC Export column name, the data type and acceptable values defined for it will automatically be used for this question.
Data type

Determines the type of field you want to use for the answer to the question.

  • Single line text – Creates a single-line text field.
  • Multi-line text – Creates a multiple-line text field.
  • Drop-down select list – Creates a drop-down list. Users can select only from the items on the list.
  • Radio buttons – Creates radio-button choices. Users can choose only one of the possible answers displayed as radio buttons.
  • Check boxes – Creates check boxes. Users can choose one or more of the possible answers displayed as check boxes.
  • Date – Creates a date field. A calendar icon is created automatically to help users find and enter a date.
  • Phone number – Creates a phone number field. The answer must be a valid phone number entered in one of the following formats: 9995551212 or 999-555-1212.
Close Closes the page without saving any of the details of the question that you have just entered.

 

Enter the details for the question as follows:

  1. Enter the text of the question in Question text.
  2. Enter a short name for the question in Short name.
  3. Leave the default setting, Yes, for Is active? because you want this question to appear on the form.
  4. Choose a Style, either Horizontal or Vertical.
  5. Enter explanatory text in Help text to help users know what this question is trying to discover.  
  6. If you know which CDC column you want this question connected to, choose it, otherwise, leave CDC Export Column blank. The CDC Export Column field appears only for specific diseases.
  7. Select the Data type you want the answer to the question to be.
  8. Click Create. The question is added to the Default View. Review the question you just defined. You can see the question text, the short name and the data type for the answer.

 

Several options appear on the right side of the page. They are described in the following table.

 

Option Description
Edit Allows you to edit the question information that you just entered. Note: You cannot change the data type of the question. You must delete the question and define it again.
Add follow up Allows you to add a questions to the form when the answer to the question is Yes. This is called a follow up question and consists of two parts: 
  • The follow up condition
  • The question or questions that appear when the follow up condition is met
Copy to library Allows you to copy the question to the library. In this way, you can store frequently used questions and value sets.
Add value set Allows you to store a set of possible answers to a question. The following data types do not allow a stored value set: single line text, multi-line text, date, phone number. All others require that you enter a set of values for the question.
Trash can icon Deletes the question and any value set or follow up questions. When you click this icon, the application asks you to confirm that you want to delete the question.
