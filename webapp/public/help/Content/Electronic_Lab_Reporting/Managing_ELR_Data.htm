Managing ELR Data

The Staged Messages page lets you view and manage incoming lab reports. To open the Staged Messages page, click Staging Area at the top right corner of the page.

The following illustration shows what the lab data looks like before you begin working with it. It lists:

  • patient's name and date of birth
  • type of specimen and the date it was collected from the patient
  • name of the lab that the data came from
  • status of themessage
  • date the test was run
  • LOINC Code
  • test type
  • result
  • units
  • test status
  • reference range for the test

 

Filtering the List

On the right side of the page, a drop-down list contains several selections for filtering what is listed. Select one of the following to filter the list:

  • Pending - these messages are waiting to be processed by a user.
  • Assigned - theses messages have been assigned to an event by a user.
  • Discarded - these messages have been rejected by a user and can no longer be processed.
  • Unprocessable - these message cannot be processed because something went wrong while processing the message.

Processing the Message

Review the message to be sure that it is useable. If a message is a duplicate or if your organization does not want to process this message any further, you may want to remove it. If you do not want to use a message, click Discard. Discarded messages are still visible when you select Discarded from the Status drop-down, but you cannot undelete them.

If you determine that you can use the incoming lab results, click Similar Events to perform a search for events in the CSI TriSano™ database that closely match the name and birth date of the message. If any of the events in the database match the name and birth date closely enough, they appear on the results list as shown in the following sample illustration.

 

 

There are several actions you can take from this list:

  • Show - Allows you to view the event for the patient listed in this row.
  • Assign lab result - Assigns the lab result to the event listed in this row.
  • Assign to new CMR using this person - Creates a new CMR for this person and assigns the lab result to it. This is useful when you already have a CMR defined for the person but this new lab result is for a different disease event. The application uses the address for the patient that is found in the existing CMR, not from the lab report.
  • Create a CMR from this message - Creates a new CMR, for a new person.
