How you store and share documents depends on your need for version control and the technical level of your project members.

To choose the best way of handling documents for your project, follow these steps:

  1. Do you need to keep your documents under version control?
    • If yes, go on to step 2.
    • If no, use the simple Documents & Files feature. On the Project Tools page, click Documents & Files.
  2. Are your project participants mostly developers working with code?
    • If yes (for example, if your project is a software development project), instruct your members to install a Subversion client and use it to connect to the project's Subversion repository. This option gives your participants the most powerful, flexible access to source control.
    • If no (for example, if your project members are business users whose output is mostly non-technical documents), add a Documents component to one or more of your project pages. Click Edit this page ⇒ Add component to page. In the Component type field, select Documents.

Note: Whichever document storage method you choose, you can link your page content to your project's documents. For help on embedding document links in page content, see the Project FAQ.
